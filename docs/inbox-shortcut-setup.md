# Apple Shortcut Setup — Inbox Capture

This guide walks through creating the iOS Shortcut that logs interactions to `inbox.txt` in iCloud Drive. The Shortcut is the only writer of `inbox.txt`; the Mac app reads and processes it.

## Prerequisites

- iCloud Drive enabled on iPhone (Settings → [your name] → iCloud → iCloud Drive → On)
- iOS Shortcuts app installed
- You know your `DATA_PATH` — by default `~/Library/Mobile Documents/com~apple~CloudDocs/NetworkingCRM`. The `inbox.txt` file lives directly inside this folder.

## Entry format the Shortcut must produce

The Mac parser expects this exact format. Each entry is delimited by `---` on its own line:

```
---
id: a3f7c2b1
date: 2026-06-28T15:30:00-07:00
contact: Alice Smith
type: meeting
location: Blue Bottle, Oakland
summary: Discussed Q3 roadmap.
Voice dictation can span multiple lines here.
---
```

Field rules:
- `id` — 8 lowercase hex characters (auto-generated by the Shortcut)
- `date` — ISO 8601 with your timezone offset (auto-generated by the Shortcut)
- `contact` — the person's full name as stored in the CRM (case-insensitive, but full name is safer than nickname unless you've set a `preferredName`)
- `type` — one of: `meeting`, `call`, `email`, `message`, `other` (defaults to `meeting` if omitted or misspelled)
- `location` — optional; physical place or video platform ("Zoom", "Blue Bottle, Oakland")
- `summary` — optional; free text, can span multiple lines

## Step-by-step Shortcut construction

Open the **Shortcuts** app on your iPhone. Tap **+** to create a new Shortcut. Name it "Log Interaction."

Add the following actions in order:

### 1. Generate the entry ID

Add action: **Generate GUID**

Then add action: **Get Text from Input** → select the GUID variable → set **Get** = "First Characters" → **Count** = 8

Store as: `Entry ID`

### 2. Capture the current date with timezone offset

Add action: **Format Date**  
- Date: **Current Date**  
- Format: **Custom**  
- Custom format string: `yyyy-MM-dd'T'HH:mm:ssXXXXX`

> The `XXXXX` token produces the timezone offset like `-07:00`. This is required by the Mac parser.

Store as: `Entry Date`

### 3. Ask for the contact name

Add action: **Ask for Input**  
- Prompt: "Who did you interact with? (use their full name)"  
- Input Type: Text  

Store as: `Contact Name`

> **Voice tip:** change Input Type to "Dictation" to use your voice. Say the full name clearly.

### 4. Choose the interaction type

Add action: **Choose from List**  
- Items (add each on a new line):  
  ```
  meeting
  call
  email
  message
  other
  ```
- Prompt: "Type of interaction"  
- Select Multiple: Off  
- Select All Initially: Off  

Store as: `Interaction Type`

### 5. Ask for location (optional)

Add action: **Ask for Input**  
- Prompt: "Location? (leave blank to skip)"  
- Input Type: Text  
- Allow Empty: On  

Store as: `Location`

### 6. Ask for summary (optional)

Add action: **Ask for Input**  
- Prompt: "Summary? (leave blank to skip)"  
- Input Type: Text  
- Allow Empty: On  

> **Voice tip:** change Input Type to "Dictation" for hands-free capture. Speak as long as needed — the Mac parser handles multi-line summaries.

Store as: `Summary`

### 7. Assemble the entry text

Add action: **Text** (the plain Text action, not "Get Text")

Type the following template exactly, inserting variables where shown (tap a variable name to insert it as a token):

```
---
id: [Entry ID]
date: [Entry Date]
contact: [Contact Name]
type: [Interaction Type]
```

Then add a **Scripting → If** block to conditionally add `location:` only when Location is non-empty:

- **If** `Location` **has any value**  
  - Add **Text** action: `location: [Location]`  
  - Add to the assembled text  
- **End If**

Similarly for summary:

- **If** `Summary` **has any value**  
  - Add **Text** action: `summary: [Summary]`  
- **End If**

Close the block with a final Text action containing just `---`.

**Combine Text** action (or use a single multi-line Text block and concatenate conditionally) — combine all pieces with **New Lines** separator into one variable `Entry Block`.

> **Shortcut implementation note:** Shortcuts' "Text" action supports multi-line content and variable insertion. Build the block in one Text action, using If blocks to conditionally include optional fields before the closing `---`.

### 8. Append to inbox.txt

Add action: **Append to File**  
- File Path: `NetworkingCRM/inbox.txt`  
- Storage: iCloud Drive  
- Make New Line: **Off** (the entry block already ends with a newline)

> The file path is relative to the root of iCloud Drive. If your `DATA_PATH` uses a different folder name, adjust accordingly.

## Complete example output

When all fields are filled in, the Shortcut appends exactly this to `inbox.txt`:

```
---
id: a3f7c2b1
date: 2026-06-28T15:30:00-07:00
contact: Alice Smith
type: meeting
location: Blue Bottle, Oakland
summary: Discussed Q3 roadmap.
Alice is excited about the Q4 partnership proposal.
---
```

When location and summary are skipped:

```
---
id: b4c8d2e1
date: 2026-06-28T16:45:00-07:00
contact: Bob Jones
type: call
---
```

## Installing the Shortcut on your iPhone

- Build the Shortcut on your Mac in the Shortcuts app (the same steps work on macOS Shortcuts)
- **AirDrop** it to your iPhone: right-click the Shortcut → Share → AirDrop → your iPhone
- On iPhone: accept the AirDrop and tap "Add Shortcut"

Alternatively, build it directly on iPhone using the steps above.

Add the Shortcut to your Home Screen for one-tap access: long-press the Shortcut in the Shortcuts app → "Add to Home Screen."

## Verification

After running the Shortcut once:

1. Open the **Files** app on iPhone
2. Navigate to: iCloud Drive → NetworkingCRM
3. Tap `inbox.txt` — you should see your entry with the `---` delimiters and all fields

On your Mac, open the CRM and press "Process inbox" to ingest the entry.

## Tips

- **Full name vs nickname:** the Mac parser matches `contact:` against the stored contact's `name` and `preferredName` fields (exact, case-insensitive). If you want "Bob" to auto-match "Robert Jones," set that contact's `preferredName` to "Bob" in the CRM first.
- **Dictation for long summaries:** the multi-line summary block means you can speak freely. The Shortcut captures everything until you stop dictating.
- **Offline capture:** the Shortcut works without a network connection. iCloud syncs `inbox.txt` to your Mac the next time the iPhone has internet access.
- **Rapid logging:** if you log several interactions in sequence, each run appends a new block — all entries are processed together when you press "Process inbox."
